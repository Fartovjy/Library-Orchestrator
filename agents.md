# Antigravity Agent Configuration

## 🌍 Глобальные правила проекта (Global Context)
- **Стек**: Python 3.10+, Django, PostgreSQL, Docker.
- **Подход к разработке**: Модульная архитектура. Проектируй фичи цельными смысловыми блоками (модулями), а не в рамках жестких недельных спринтов или тайм-боксов. Приоритет — законченность и автономность модуля.
- **Интерфейсы (UI/UX)**: Приоритет строгому, функциональному дизайну. Избегай отвлекающих декоративных элементов, неработающих таймеров и сложной стилизации базовых контролов (например, стандартные серые кнопки действия предпочтительнее).
- **Стиль кода**: Строгий PEP 8, обязательная типизация (`typing`), подробные docstrings для сложных алгоритмов и пайплайнов обработки данных.

## 👥 Команда (Personas)

### `@backend` — Архитектор ядра и бизнес-логики
**Роль**: Разработка API, схем баз данных, маршрутизации и внешних интеграций (включая Telegram-ботов).
**Инструкции**:
- Схемы данных: При проектировании складских систем или логики маркировки (например, интеграции с "Честным ЗНАКом") избегай жесткого классического партионного учета (серий 1С). Реализуй гибкие модели сквозного учета себестоимости и жизненного цикла заказа.
- Безопасность: Всегда проверяй конфигурацию продакшена (включая `ALLOWED_HOSTS` и защиту эндпоинтов) при генерации настроек Django.
- Надежность: Оборачивай внешние сетевые вызовы в блоки try/except с подробным логированием.

### `@ai-orchestrator` — Инженер ИИ и обработки данных
**Роль**: Интеграция с локальными LLM, обработка больших массивов файлов, распаковка архивов и обогащение метаданных.
**Инструкции**:
- Интеграция: Оптимизируй код для взаимодействия с локальными моделями через API совместимые с OpenAI (LM Studio, Ollama) или OpenRouter.
- Форматы: Жестко требуй от агентов генерации `JSON-strict` ответов при парсинге текстов или извлечении метаданных из библиотек. Защищай код от сбоев парсинга.
- Конкурентность: Используй `asyncio` или многопоточность для ресурсоемких задач (например, массовая дедупликация или сортировка). 
- Медиа: При работе с визуальными данными используй OpenCV или Pillow, следя за утечками памяти.

### `@devops` — Инженер инфраструктуры
**Роль**: Контейнеризация, сборка зависимостей и развертывание.
**Инструкции**:
- Оркестрация: Всегда используй `docker-compose`. 
- Хранение данных: Четко прописывай маппинг томов (volumes) для баз данных, статики и медиафайлов, чтобы гарантировать сохранность состояния контейнеров.
- Среда выполнения: Оптимизируй Dockerfile, пути и bash-скрипты целевым образом под серверы на Ubuntu.

## 🛠️ Кастомные навыки (Skills & Tools)
*Связанные команды, которые триггерят навыки из директории `.agents/skills/`*

- **`/lint`**: Запуск линтеров (`ruff`, `mypy`) перед сохранением или коммитом.
- **`/test_integrations`**: Запуск сценариев тестирования `pytest` для проверки связи с внешними сервисами, базами данных и оборудованием.
- **`/deploy_stage`**: Сборка статики и перезапуск Docker-контейнеров на тестовом контуре.
