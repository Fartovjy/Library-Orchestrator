# Оптимизация I/O и снижение нагрузки на HDD при сортировке библиотеки

## 1. Архитектурный рефакторинг приложения (Оркестратор и Агенты)
- [ ] **Отказ от временных файлов при чтении/распаковке**
  - [ ] Перевести чтение архивов и файлов (FB2, PDF и др.) в оперативную память.
  - [ ] Внедрить использование `io.BytesIO` для передачи контента агентам на анализ без сохранения промежуточных файлов на диск.
- [ ] **Внедрение пакетного (Batch) перемещения файлов**
  - [ ] Создать пул/очередь в оркестраторе для накопления путей отсортированных файлов.
  - [ ] Настроить триггер пакетного перемещения (например, при достижении лимита в 500-1000 файлов или по таймауту).
  - [ ] Заменить единичные вызовы `shutil.move` или `os.rename` на последовательную обработку очереди в отдельном потоке.
- [ ] **Асинхронность дисковых операций**
  - [ ] Изолировать I/O операции в отдельный асинхронный поток/процесс (например, с использованием `asyncio` или `ThreadPoolExecutor`), чтобы агенты не простаивали в ожидании ответа от диска.
  - [ ] Настроить упреждающее чтение (read-ahead) для формирования очереди задач для агентов.

## 2. Настройка окружения и операционной системы (Windows)
- [ ] **Оптимизация NTFS**
  - [ ] Отключить запись времени последнего доступа. Открыть CMD от имени администратора и выполнить: `fsutil behavior set disablelastaccess 1`.
  - [ ] Проверить, что галочка «Разрешить кэширование записей для этого устройства» включена в диспетчере устройств для целевого HDD.
- [ ] **Исключения защитника Windows / Антивируса**
  - [ ] Добавить корневую папку исходной библиотеки в исключения сканирования в реальном времени.
  - [ ] Добавить целевую структуру папок (куда складываются отсортированные книги) в исключения.
- [ ] **Отключение индексирования**
  - [ ] В свойствах исходной и целевой папок снять галочку «Разрешить индексировать содержимое файлов в этой папке в дополнение к свойствам файла».

## 3. Оптимизация базы данных (если используется SQLite/локальная БД)
- [ ] **Настройка прагм (PRAGMA)**
  - [ ] Включить режим WAL (Write-Ahead Logging) для снижения блокировок: `PRAGMA journal_mode = WAL;`.
  - [ ] Увеличить кэш страниц в памяти: `PRAGMA cache_size = -64000;` (~64 МБ).
  - [ ] Настроить синхронизацию: `PRAGMA synchronous = NORMAL;` (баланс между скоростью и надежностью).
- [ ] **Пакетные транзакции**
  - [ ] Обернуть операции `INSERT`/`UPDATE` статусов распознавания в крупные транзакции. Делать `commit()` каждые 500-1000 записей, а не после каждого файла.

## 4. Интеграция RAM-диска (Пайплайн высокой интенсивности)
- [ ] **Подготовка среды**
  - [ ] Установить софт для создания виртуального диска в оперативной памяти (например, ImDisk).
  - [ ] Выделить объем RAM-диска (например, 2–4 ГБ в зависимости от доступной памяти системы).
- [ ] **Разработка конвейера перекачки (HDD -> RAM -> HDD)**
  - [ ] Написать скрипт/модуль оркестратора для копирования "чанков" (порций несортированных файлов) с HDD на RAM-диск одним последовательным потоком.
  - [ ] Перенастроить пути оркестратора и агентов: рабочая директория для распаковки, парсинга и работы LLM должна находиться строго на RAM-диске.
  - [ ] Написать функцию возврата: после успешной сортировки чанка на RAM-диске, оркестратор пакетно переносит готовые файлы в целевые директории на HDD.
