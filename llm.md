## Почему qwen2.5:7b

Alibaba Qwen 2.5 7B — лучший выбор среди доступных локальных моделей для задач с русскоязычными книгами:

- **Русский язык**: Qwen 2.5 обучен на масштабном русскоязычном корпусе, стабильно извлекает кириллические имена авторов и названия без транслитерации.
- **VRAM**: 4.7 GB — помещается в RTX 2060 (8 GB) с запасом ~3 GB, A6 и A7 не вытесняют модель из памяти при параллельной работе.
- **Скорость**: 7B параметров → быстрый вывод, важно при обработке сотен файлов.
- **JSON**: хорошо следует структурированным инструкциям, стабильно возвращает валидный JSON.

## Сравнение доступных моделей (ollama list)

| Модель | Размер | Русский | VRAM RTX 2060 | Вывод |
|--------|--------|---------|---------------|-------|

| `gemma4-ablit:12b` | 7.6 GB | хороший | ⚠️ впритык (~0.4 GB) | Альтернатива для качества |

| `ibm/granite3.3:8b` | 4.9 GB | слабый | ✅ | Не для русского |
| `granite-code:8b` | 4.6 GB | нет | ✅ | Только код |

### Если нужно ещё лучшее качество

Используйте `gemma4-ablit:12b` для A6 — 12B параметров дают более точное извлечение метаданных.
Но тогда A6 и A7 будут использовать разные модели → Ollama будет выгружать/загружать модели при переключении.
**Рекомендация**: одна модель для обоих агентов.

## Настройки воркеров для Ollama

```python
LM_WORKERS = 1        # GPU обрабатывает 1 запрос за раз
RENAME_WORKERS = 1    # та же GPU, та же модель
```

Увеличение LM_WORKERS выше 1 при локальном Ollama не ускоряет работу — GPU всё равно выполняет запросы последовательно, а очередь создаёт лишнюю нагрузку на память.

## Переход на облако (OpenRouter)

Если местная модель даёт плохое качество для сложных книг — OpenRouter бесплатные модели:

```python
LM_URL = 'https://openrouter.ai/api/v1/chat/completions'
LM_MODEL = 'google/gemma-3-27b-it:free'   # или 'meta-llama/llama-4-scout:free'
LM_API_KEY = 'sk-or-v1-...'
LM_WORKERS = 1   # бесплатный лимит ~20 запр/мин
```
