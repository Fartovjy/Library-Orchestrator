# Library Orchestrator

## Интеллектуальная разборка и каталогизация большой книжной библиотеки

`Library Orchestrator` — это приложение для длительной, безопасной и возобновляемой обработки большой файловой библиотеки, в которой книги лежат вперемешку с архивами, папками, техническими файлами, изображениями, видео и служебным мусором.

Проект создан для сценария, где обычный “разовый сортировщик” уже не справляется:

- файлов очень много;
- библиотека собиралась годами;
- часть книг лежит в архивах;
- часть книг состоит из нескольких связанных файлов;
- места на диске мало;
- обработка занимает не часы, а дни;
- классификация по одному имени файла часто даёт плохой результат.

Идея проекта в том, чтобы превратить хаотичную файловую свалку в структурированную библиотеку, не разрушая содержимое и не теряя логические связи между файлами.

---

## Что делает приложение

`Library Orchestrator` проходит по библиотеке постепенно и для каждого объекта решает, что с ним делать:

1. Определяет тип файла или контейнера.
2. Отделяет мусор и неподходящие файлы.
3. Распаковывает поддерживаемые архивы и папки во временную рабочую область.
4. Пытается распознать книгу, автора, название и жанр.
5. При необходимости передаёт объект на более глубокий анализ.
6. Пересобирает результат в единый `ZIP`-архив.
7. Вычисляет хэши и отслеживает дубликаты.
8. Перекладывает результат в целевую структуру библиотеки.
9. Сохраняет состояние так, чтобы работу можно было продолжить после остановки или сбоя.

В результате на выходе формируется понятная библиотечная структура:

`Library / Жанр / Буква автора / Автор / Автор - Название.zip`

Дополнительно создаются отдельные ветки:

- `_Duplicates`
- `_Manual_Review`
- `_Trash`
- `_Damaged`

---

## Для кого это приложение

Проект особенно полезен, если у вас:

- огромная домашняя библиотека книг и архивов;
- старый файловый архив “на потом”;
- коллекция технической, художественной и учебной литературы без единой системы;
- библиотека на HDD, где важно не перегружать диск;
- задача, которую нужно решать постепенно и аккуратно, а не одним разрушительным проходом.

## Ключевые особенности

### 1. Работа на больших объёмах

Приложение не предполагает, что библиотеку можно полностью распаковать или обработать за один запуск. Оно рассчитано на долгую поэтапную работу с сохранением состояния.

### 2. Поддержка локальных LLM через LM Studio

Для распознавания жанра, автора и названия можно использовать локальные модели через LM Studio. Это особенно полезно там, где имя файла само по себе ничего надёжно не говорит.

### 3. Защита структуры книги

Если книга состоит из нескольких файлов, приложение не разрывает её на части. Оно стремится сохранить комплект вместе и привести его к единому нормализованному архиву.

### 4. Дедупликация

Каждый результат может быть проверен по хэшу. Это позволяет выносить повторяющиеся экземпляры в отдельную ветку `_Duplicates`.

### 5. Возобновляемость

Процесс можно остановить и продолжить позже. Для долгих библиотечных задач это критично.

### 6. Фильтрация мусора

Проект умеет отсеивать не только по расширению файла, но и по сигнатурам первых байтов (`magic bytes`). Это помогает раньше отделять изображения, видео, служебные бинарники и ложные контейнеры.

### 7. Терминальный интерфейс для долгих прогонов

Во время работы показывается живой экран состояния:

- сколько уже обработано;
- сколько осталось в текущем батче;
- какой файл обрабатывается сейчас;
- какой агент работает;
- сводка по статусам;
- безопасные горячие клавиши остановки.

## Как устроена система

Архитектурно проект построен как оркестратор с агентами.

Основные роли:

- `Unpack Agent` — распаковка и подготовка содержимого;
- `Archivarius` — быстрый анализ и первичная классификация;
- `Expert` — глубокий анализ сложных случаев;
- `Pack Agent` — упаковка результата в единый архив;
- `Placement Agent` — перенос в целевую библиотеку.

Центральный `Orchestrator` управляет:

- порядком обхода;
- сохранением состояния;
- throttling по нагрузке на диск;
- маршрутизацией в нужные ветки;
- безопасной остановкой и продолжением работы.

---

## Почему проект полезен на практике

Большинство сортировщиков файлов работают по очень простым правилам:

- имя файла;
- расширение;
- одноразовый запуск;
- отсутствие восстановления после сбоя;
- нет различия между книгой, архивом и мусором.

`Library Orchestrator` решает задачу глубже:

- понимает, что библиотека может быть многослойной;
- умеет работать с архивами и контейнерами;
- не требует полного распаковывания всей коллекции;
- поддерживает модель “сегодня обработали часть, завтра продолжили”;
- допускает гибридный режим: быстрые правила + локальная LLM;
- сохраняет контроль над процессом даже на очень больших объёмах.

## Текущий статус проекта

На текущем этапе проект уже умеет:

- читать конфиг и рабочие пути;
- поддерживать отдельную папку результата;
- вести SQLite state store;
- распознавать часть типов файлов по сигнатурам;
- маршрутизировать объекты в `Library`, `_Trash`, `_Manual_Review`, `_Damaged`;
- использовать LM Studio для fast/deep классификации;
- показывать терминальный дашборд;
- выполнять безопасную остановку через hotkeys и stop-файл.

Это уже рабочая база, но проект всё ещё развивается. В ближайших итерациях особенно важны:

- более точное распознавание книжных форматов;
- улучшение качества жанровой классификации;
- тонкая работа с многотомными и сложными архивами;
- параллельные воркеры;
- более строгая дедупликация;
- улучшенная очистка и нормализация старых библиотек.

## Запуск

Пример запуска:

```powershell
cd E:\Разобрать\orchestrator_project
python -m orchestrator.main --config config.example.json run --limit 50
```

Безопасная остановка:

```powershell
python -m orchestrator.main --config config.example.json stop
```

Горячие клавиши в активном окне:

- `Ctrl+X`
- `Esc`
- `Q`

---

## Философия проекта

Этот проект не про “быстро разложить файлы хоть как-нибудь”.

Он про другое:

- аккуратно работать с большими массивами данных;
- не разрушать библиотеку;
- уметь останавливаться и продолжать;
- сохранять контроль;
- использовать локальный ИИ как инструмент, а не как единственный источник решений.

Если коротко, `Library Orchestrator` — это инструмент для тех, кто хочет не просто перебросать файлы по папкам, а постепенно превратить хаотичный архив в настоящую библиотеку.
